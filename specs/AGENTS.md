# Specs

Criar spec ANTES de implementar nova feature.

## Formato

```
# SPEC: [Nome da Feature]

## 1. Visão Geral
// Descrição breve do que será implementado

## 2. Análise de Opções (se aplicável)
// Comparação de libs/abordagens em tabela

## 3. Decisões Técnicas
// Bibliotecas escolhidas, motivações

## 4. Estrutura de Arquivos
// Tree da nova estrutura

## 5. To-Dos
- [ ] Tarefas de implementação

## 6. Dependencies
// Dependências necessárias (npm install)
```

## Regras

- Uma feature = um arquivo `.md`
- Nome: `kebab-case` (ex: `code-editor.md`, `drizzle.md`)
- To-Dos com checkboxes para acompanhar progresso
- Incluir exemplos de código quando relevante
- Decisões com justificativas curtas
